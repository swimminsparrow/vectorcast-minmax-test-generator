4. Generazione dei Test di Boundary

Dopo aver terminato la configurazione dei parametri, selezionare “Run → Build Boundary TestCases”.

Nel Default Case (senza alcuna opzione specificata) saranno generati 4 Test Script (.tst):

  • NOMEENVIRONMENT_MAX.tst: contiene il MAX per ogni variabile specificata dall’utente, o in caso contrario il MAX rappresentabile per tipo.

  • NOMEENVIRONMENT_MIN.tst: contiene il MIN per ogni variabile specificata dall’utente, o in caso contrario il MIN rappresentabile per tipo.

  • NOMEENVIRONMENT_Intorno_Destro.tst: per ogni variabile specificata dall’utente (MAX o Locked Value), il valore viene incrementato di 1.

  • NOMEENVIRONMENT_Intorno_Sinistro.tst: per ogni variabile specificata dall’utente (MIN o Locked Value), il valore viene decrementato di 1.

I Test Script generati sono salvati nella cartella VectorCastEsterMinMaxGenCacheFolder.

Opzioni:

  • Build Combination TestCases: se abilitato, per ogni funzione verrà generato un test di intorno partizionato per ogni variabile per cui è stato specificato un MIN, MAX o Locked Value.

  • Import TestCases in VectorCast (richiede Server Licenze VectorCast avviato): se abilitato e se è stato inserito un Path contenente un Environment valido, gli script generati saranno importati automaticamente nell’ambiente VectorCast specificato.

  • Execute TestCases in Environment (richiede Server Licenze VectorCast avviato): se abilitato e se è stato inserito un Path contenente un Environment valido, i TestCase importati nell’ambiente VectorCast saranno eseguiti automaticamente al termine.
